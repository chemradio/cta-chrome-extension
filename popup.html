      Output

      Flattens captures onto a white background and strips the alpha channel and colour-profile chunks. Fixes Adobe ScriptUI / Direct2D drawbot panels that reject the screenshots (HRESULT 0x80070057).

      How capture works

      Capture this post / story

      When you open the popup on Facebook, Instagram, Telegram (t.me), X / Twitter or VK, the extension checks whether the page contains a post or a story. If it does, a "Capture this post" (or "story") button appears at the top. Click it to remove comments, sidebars and other clutter around the target and capture just that element.

      Auto Capture

      One-click cleanup + full-page capture: runs the ad-remover and per-site cleanup, then captures the whole page at the chosen scale. Use this when you want the whole page, not a single post.

      Page Capture

      Captures the whole page at a chosen resolution. Pick a preset:

      • User — the current tab's viewport size (default). Combined with the 2× quality multiplier, gives you a sharp shot at the exact size you see in the browser.
      • Full Page — viewport width, full document height (measured live).
      • Vertical — viewport width × (width × 3.5), good for long social posts.
      • FullHD — 1920 × 1080.
      • 4K — 3840 × 2160.
      • Custom — your own width × height.

      The page is emulated at this resolution before capture, so layouts adapt as if the window were that size.

      Capture Element

      Click the button, then hover any element on the page. A cyan glow shows what will be captured. Use the scroll wheel to change depth (up = parent, down = first child). Click to commit.

      Quality multiplier

      1× / 2× / 3× / 4× — scales pixel density. 2× is the default and works well for most uses. 3×–4× produces large but very sharp images suitable for print or motion graphics.

      Helpers

      Cleanup — runs the ad-remover and per-site cleanup once on the current page, without taking a screenshot. Useful before Page Capture if Auto isn't the right mode.

      Remove Elements — manual click-to-remove mode. Hover targets, scroll to change depth, click to delete. ESC to stop. Removed selectors are saved as your personal filters (see Expert mode for managing them).

      How filters work

      Three filter tiers

      When you run Cleanup or Auto Capture, three sources of filters are merged:

      • EasyList — the standard adblock cosmetic filter list. Refreshed from easylist.to automatically (weekly).
      • Bundled filters — hand-curated selectors shipped with the extension for sites the dev has tuned. Updated when the extension updates.
      • User filters — selectors you collect, either via "Remove Elements" (which auto-saves each click) or by typing them in here.

      Duplicates between tiers collapse automatically, so promoting a user filter to the bundled list is non-disruptive.

      Adding filters manually

      Paste a CSS selector into the input, or paste a fragment like class="foo bar" or <div data-testid="ad"> — it gets converted to a valid selector automatically. Preview shows the normalized form before adding.

      • This host — applies only on the current domain (and subdomains).
      • All hosts — applies everywhere. Use sparingly.
      • Target parent element — wraps the selector in *:has(> …) so the parent gets removed instead of the matched element. Good for "remove the card that contains this badge".
      Managing your filter list

      Filters for the current host are listed under the host name; global filters under "All hosts". Click × on a row to remove that one. The Clear Domain Filters and Clear Global Filters buttons wipe entire tiers (EasyList and bundled are untouched).

      Disable bundled filters turns off the dev-curated tier — useful for debugging which tier is causing unwanted removals.

      Exporting filters

      Export Filters downloads your collected user filters as a JSON file (shape: {global, domains}). Email this to the extension dev — selectors worth sharing get merged into the bundled list and propagated to everyone in the next update.
